Registry: Hierarchical Namespaces

June 4, 2017 classes

Registry class.

This class make its possible to store objects and retrieve them, as done with classes such as Pdef, Ndef, OSCdef etc, but without having to subclass. To create this behavior instead, add a method to Symbol for accessing the kind of object needed. For example: \aSymbol.oscFunc. This also allows easier customization of how the object is created if not found.

Store an object in Library under an array of objects (called a path). When accessing the path, create the object if not already stored, else return the previously stored object.
